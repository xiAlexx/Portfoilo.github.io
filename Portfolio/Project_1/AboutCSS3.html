Introduction to CSS3

Cascading Style Sheets (CSS) is the language web page authors use to tell a browser or other user agents how to format an HTML document. Style Sheets provide precise control over how structural elements appear on a web page. A single style sheet can be used to provide styles for an entire page or multiple pages on a site. For example the style of this web page about CSS3 is the same as the the web page for HTML if I was to make a change on my style it would change every website I have linked that CSS file to.

Older presentational HTML is a thing of the past, the "Table" element for example should not be used to simply arrange items in a grid as in the past. The "Font" element is no longer a part of the HTML specification. Finally in HTML5 presentational elements have either been redefined or have been deprecated (many not support broswers) and their use should be avoided.

The Advantages Of Style Sheets

What are the advantages to using style sheets?
  • More flexibility.
  • HTML5 no longer includes display-oriented or "presentational" elements and attributes meaning these may cease to be supported by browsers in the near future.
  • Style sheets control every aspect of page display.
    1. Space between lines.
    2. Character spacing.
    3. Page margins.
    4. Positioning of elements.
  • Apply changes globally.
  • Instruct broswers to control appearange.
  • Create dynamic pages using animations.

Some Simple Steps To Styling

  1. Select the elements in a document that you want to style. (selectors)
  2. Write declarations that apply to the selectors.
    • Property name and value
    • Font-family:Verdana
    • Font-weight:Bold
    • Padding: 10px
  3. Link the style rules and markup.
  4. The browser handles the rest.

Discovering Cascading Style Sheets

We understand "style Sheets" but what's with the term cascading?
Well cascading is the process browsers use to determine which style to apply to an element.

What if there is a conflict and two style sheets apply a different style rule to the same element?
The last applied style will be the winner.
All web broswers follow the same rules which developers need to know.

Types Of Selectors And Syntax

    Selectors
  • The universal selector (*) - matches any element and is often used for global properties.
  • *(margin:0px)
  • The most blunt selector there is, extremely powerful but not specific

    • The element type selector
  • Select a particular element such as h1, p, a, div etc...
  • Works the same as universal but with a single element

  • #container(background-color:orange;)

  • Specifies certain instances of an HTML markup element.
  • To use you combine the HTML markup element wuth the global "class" attribute. Then write CSS style rules to target a class prefacing the class name with a (.) For example:
  • .Warning(color:red;)
  • H1.warning(color:red;font-weight:bolder;)

    • Selecting Multiple Elements
  • Multiple elements can easily be selected.
  • Div, p, ul {...}

    • Selecting Child Elements
  • If you want to select all the "p" elements inside a "div" element or other child relationship.
  • Div p {...}

    • Selecting By Attribute
  • You can select an element based on the specific content of an attribute.
  • A [href=http://www.google.com]{...}

    • Selecting Pseudo-Classes

  • The Most common pseudo class are those for the "a" element.
  • Link.(unvisted)
  • Visted.
  • Hover.
  • Active.(when the link is clicked)
  • A:hover{...}

    • Selecting Pseudo Elements
  • Refers to a specified part of an element such as the first letter or the first line of the element
  • "p" This is some text "/p"
  • You can select the letter T using this selector
  • p::first-letter {...}

    • Case Insensitivity
  • For the most part CSS selectors and rules are case-insensitive.
  • Div {Background-Color:GREEN;}
  • DIV {background-coloR:green;}
  • The only expection is when selecting classes and ids which ARE case-sensitive.

    • Order of Precedence
  • A very important concept that can be frustrating if you are new to CSS.
  • If you are struggling with rules not being applied check these:
  • Case senitivity for classes and ids.
  • Appropriate class values for elements.
  • Precedence.
  • Understanding the rules for precedence.
  • If no style is specified, elements inherit styles from their parent container or use the broswer's default styling.

Attributes

Understanding The CSS Box Model

This section will explain what the box model is, height and Width, Margin and padding and finally working with borders.

What Is The Box Model?

All elements in CSS are treated as a box essentially wrapping around ever single HTML element on the page. The box consists of four properties, Margin, Borders, Padding and Content.

Height And Width

The height and width properties of the box deal with the dimensions of the content area and are given as a # of pixels or percentage.

  • Height:100px;
  • Height:80%;
  • Height:auto

If the amount of content exceeds the size of the box surrounding its element the content will flow out of the box and result in a badly formatted web page. Use the overflow property to prevent this.

  • Visible
  • Hidden
  • Scroll
  • Auto

Margin and Padding

The padding controls the free space around the content inside each box while the margin controls the free space around the outside of the border of the box. Both of these controls are transparent and therefore changing their color is not possible however, the width can be modified to adjust positioning of elements. Several syntax options exist for both margins and padding.

Margin and Padding Syntax
  • Margin: 25px; - sets all four margins to 25px

  • Margin: 25px 50px; - sets the top and bottom margins to 25px and the left and right to 50px

  • {margin-top: 25px; margin-right: 50px; margin-bottom: 50px; margin-left: 10px;}

  • Margin: 25px 50px 50px 10px

  • Additionally margins (not padding) can have negative values.

  • A helpful tip for center something you can use margin: 0px auto;

Working With Borders

The CSS border properties allow you to set various properties for the border of each element in the CSS box model.

  • Border-width
  • Border-color
  • Border-style
  • Border-radius

The border-width is the thickness of the border that surrounds each HTML element. This is normally set in pixels and provides the alternative of using one of the three predefined values.

  • Thin
  • Medium
  • Thick
Uses the same syntax as padding and margins.
  • Border-width: 25px - sets all borders to 25px.

  • Border-width: 25px thin - sets all the top and bottom to 25px and the left/right to thin.

  • Border-top-width: 30px.

Border-color is used to configure the color of the border.

  • Use one of the 140 predefined color names

  • Use RGB notation - RGB (red, green and blue) refers to a system representing the colors used on a digital display screen. Red, green and blue can be combined in various proportions to obtain any color in the visible spectrum.

  • RGB (0,255,0) makes all green or RGB (255,0,255) makes all red and all blue which makes a purple/pink

  • Use hexadecimal notation - uses 6 digits to represent color

  • 00 00 00

  • FF 00 00

  • FF 00 FF

  • Set to transparent

Border style is used to configure the various border styles which are available in CSS.

  • None
  • Dotted
  • Dashed
  • Solid
  • Double
  • Groove
  • Ridge
  • Inset
  • Outset

Border-radius is used to create borders with rounded corners. The values are commonly given in pixel or percentage. You can set individuallly for the four corners.

  1. Top-left
  2. Top-right
  3. Bottom-right
  4. Bottom-left

Example:

  • Border-radius: 5px

Alternatively you can simply use the "BORDER" property as shorthand specifying:

  • Width
  • Style
  • Color
  • Border: 5px solid green;
  • *border-radius cannont be confirgured using this shorthand method

Working With Page Layouts

In this section I will breifly go through some of my exprience when working with page layouts. This is will consist on 3 chapters.

Positioning Elements

As the name implies the CSS positioning property is used to configure the location of the element on the page. In case of an overlap the positioning element specifies which element should be on top. There are four separate methods that exists for positioning elements.

  1. Static Positioning
    2. Elements are positioned according to the normal flow of the page. All HTML elements use static positioning by defualt and there generally isn't a reason to specify this type of position unless you are attempting to override another style rule.

  2. Relative Positioning
    3. Relative postitioning is a method that positions an element "relative to its normal position". Normal is the default position when no positioning rule is specified and is also used when static position is in place. So if an element would normally be directly beneath the preceding element using relative positioning will change that position in comparison with the normal position. Relative positioning can also reslut in elements that overlap in which case you have to use the z-index property. Elements with the greater z-index value will be positioned in front.

  3. Fixed Positioning
    4. Fixed positioning means that an element will always stay at its assigned location and will not move even when the page is scrolled.
    Usally found in:
    • Headers
    • Social Sharing Buttons
    • Footers

    When using fixed positioning you can use the top property to specify the number of pixels the element should be from the top of the page and the left property for distance from left of the page.

  4. Absolute Positioning
    5. Absolute positioning means that an element is positioned relative to the first parent element which has a position other than static. If none is found it is positioned relative to the page. This positioning is similar to its parent instead of its normal position.

Floating Elements

Floating is a technique used to arrange elements on a page by taking them out of the normal flow. Here are some examples:

  • Float: Left; - pushes the element to the left the furthest it can go.
  • Float: Right; - pushes the element to the right the furthest it can go.

Items that are not configured to float other obbjeects will follow normal flow and flow around the floated element.

Controlling Display and Visibility

In addition to controlling the positioning of elements using position and float you can control the visibility of those elements on the page as well. The properties allow you to remove or hide elements when necessary.

  • Display - Controls how the element is displayed and has three possible values.
  • None - the element disappears from the page.
  • Inline - the element is displayed as an inline element and thus does not start and end with a new line.
  • Block - the element is displayed as a block element and started and ends with a new line.
  • Visibility - can be used to hide an element
  • Visibility: Hidden;
  • This hides the element but it is still present on the page.

Working With Colors And Backgrounds

CSS allows author to define colors in two separate ways.

  • Name
  • Number
  • RGB notation
  • Hex notation

There are 16 basic colors that are part of the CSS3 specification along with a much longer list of extended colors which are supported by today's browsers.

When using numbers either RGB or Hex notation can be used and in both cases you are choosing the appropriate mixture of Red, Green and Blue.

  • RGB notation - the first number is the amount of red, the second the amount of green, the third the amount of blue.
  • RGB (0,255,0) - pure Green or RGB (0,0,255) - pure blue
  • The actual code would be [color:rgb(0,255,0);]

  • Use hexadecimal notation - uses 6 digits to represent color but is the same where the first 2 digits are for red and so on.
  • 00 00 00
  • FF 00 00
  • FF 00 FF
  • The actual code would be {color#00FFFF}

Colors can be defined using two CSS properties

  • Color
  • Defines font color.
  • Also used to define colors for links in various states using pseudo classes.

  • Background-Color or Background
  • Defines the color of background for an entire page or for particular elements on the page.

Backgrounds

There are two options for configuring the background of an element in CSS.

Background-color
Background-image

The use of the background-color property is similar to how you specify the color for a border in CSS boxes.

  • Use the color name
  • Use rgb notation
  • Use hexadecimal notation

In order to spice things up a bit some designers utilize the background-image property instead of a standard color. CSS provides great flexibillity with how we want the image to be display. The first step is to specify the URL to the image using the background-image property.

  • Background-image:URL("imagefile.jpg");

By default the image is placed in the top-left corner of the element and will automatically repeat itself both vertically and horizontally. If this is not desirable you can use the background-repeat property to modify the default setting.

  • Repeat - the default setting repeats horizontally and vertically
  • Repeat-x - the image will only repeat horizontally.
  • Repeat-y - the image will only repeat vertically.
  • No-Repeat - the image will not be repeated.

Examples:

  • Background-repeat:repeat-x;
  • Background-repeat:no-repeat;

By default the background image will scroll with the page but this can also be configured using the background-attachment property.

  • Fixed
  • Scroll

The background-position property is very useful if you want to specify which area of the background image you would like to display.

  • This is useful when you have an image that is larger then the element for which is serving as a background.
  • If you have an image that is 200x300 and an element that is 100x100 then only a portion of the image (the top left portion) will display.
  • In order to display a different portion of the image you would need to change the position of the image.

The most common way is to specify the back ground-position value using pixels and the syntax background-position: xpos ypos.

  • The first number shifts horizontally.
  • Negative values shift the image to the left.
  • Positive values shift the image to the right.

  • The second number shifts vertically.
  • Negative values shift the image up.
  • Postive values shift the image down.

A font is a set of characters that share a similar design.

  • Times New Roman
  • Arial
  • Verdana
  • Comic Sans

Font properties in CSS are used to format the appearance of words and text on a page. The font property is concerned with how the characters look while the text property is concerned with everything else. Font families define the three major categories of font type with two additional types.

  • Serif.
  • San serif.
  • Monospace.
  • Cursive.
  • Fantasy

When specifying the font-family you should always specify multiple fonts in case the user does not have the font you specify. The font size property sets the size of the text and can be set in multiple ways.

  • Pixels (px)
  • Em
  • Percentage (%)
  • Keywords

Ems and percentage are the recommended methods because users can change font sizes in the browser!
Examples:

  • Font-size: 20px;

  • Font-size:150%
  • Font-size:large;

Font-style property is used to specify different styles.

  • Normal
  • Italic
  • Oblique

Font-weight property is used to specify bold text and works as the counterpart to font-style

  • Normal, boldm or bolder (some browsers may not support bolder)
  • Numbers 100-900 (above 600 is considered bold)

  • Embedded OpenType - an IE only font file format.
  • Web Open Font Formatt (WOFF) - not a new format but a new way to package OpenType and TrueType.

Steps to using additional font. You can find a font and purchase it. You can pay for a font service and use open source fonts. Or you can link the font using the @font-face rule.

Lists, Links And Tables

Lists are powerful tools used to group together similar elements and give site visitors an easy way to view groups of information. Lists can be created for just about any tye of content and use at least two compontents.

  1. A markup element that tells the browser a list is on the way.
  2. A markup element that tells the browser this particular item goes into a list

All lists types can be styled using CSS3, there are 3 types of lists.

  1. Ordered lists
  2. Unordered lists
  3. Definition lists

The method for styling losts is the list-style-type property which allows you to set item markers for your list. Item markers refer to the characters which are used to delineate list items. Bullets for an unordered list by default and numbers for an ordered list by default.

Using the list-style-type property for ordered list can select the following values.

  • Lower-roman - (i,ii,iii)
  • Upper-roman - (I,II,III)
  • Decimal - default
  • Decimal-leading-zero - the number with a leading zero
  • Lower-alpa - (a,b,c)
  • Lower-greek
  • Upper-alpha - (A,B,C)

Using the list-style-type property for unordered lists allows you to specify the particular shape or symbol that you want to use as a bullet. The default marker is a disc (fiiled disc) other options are Sqaure and Circle. In addition, you can disable the marker entirely using the list-style-type property with a value of none. If you do not desire any of the additional list markers you can use an image of your own as the marker. You can use the list-style-image property to specify the url of the image that you would like to use.

The list-style-position property is used to specifiy whether the list item markers should appear inside or outside the content flow. BY default list items are outside and there is a certain amount of indentation. If this is modified and set to inside the item markers are displayed after the indentation which simply results in more indentation.

Links (Using Navigation Bars)

Navigation bars are commonly created as a combination of lists and links and are used to navigate to different areas of a specific website. The individual items in a navigation bar are simply hyperlinks that use a relative link. Links can be styled using all of the style discussed thus far. Colors, Backgrounds and Pseudo classes. The first step to creating a navigation bar is to create an unordered list of items for the menu and make those items hyperlinks to other pages. Typically you would then use the list-style-type: none; declaration to remove the default bullet points. Next you would style the bullet points and then style the hyperlinks. Finally for horizontal navigation bars you would use the float: left; property.

Dynamic CSS3 Properties

Using Transforms

The transform property in CSS3 allows for advanced management of web page elements.

  • Rotation
  • Scale
  • Skew
  • Reposition

All of this while the margins are left intact which results in the rest of the page being left intact. Transforms may not be supported by every browser that accesses your site so you should check for support for implementing on your site.

Two types of transforms exist:

  1. 2D
  2. 3D

2D transforms operate solely on the X and Y axis that traditional web development has been confined to and all major web browsers support this type of transform.

Types of transform that will change the object display are:

  • Translate - repositions an object
  • Rotate - rotates an object
  • Scale - stretchs an object in one of the avaliable directions
  • Skew

All of these options are accessed through a single style property known as transform. In the early days additional properties were required and are still ideal for your coding to ensure support:

  • -moz-transform: - Firefox
  • -webkit-transform: - Google chrome, Safari
  • -ms-transform: - IE
  • -o-transform: - opera

The translate value of the transform property will reposition the object. By default translate will occur to the right as well as on the X axis by default.

  • -webkit-transform:translate(90px);
  • To be explicit you can use translateX or translateY or specift both
  • -webkit-transform:translate(100px, 50px)

The scale property of the transform style is used to stretch or compress the element on our page. Just like translate the default for scale is the equivalent of scaleX but you can use scale Y when needed. Scale does not use the PX notation but instead uses a fraction amount.

  • -webkit-transform:scaleX(2.0);
  • -webkit-transform:scaleX(2.0)scaleY(2.0);
  • -webkit-transform:scaleX(2.0)scaleY(2.0)translate(100px, 50px);

In addition to 2D transforms 3D transforms are supported by most browsers currently and a part of the CSS3 speccification. There are several steps and values which are used to achieve 3S rotation.

  • rotateY
  • rotateX
  • rotate
  • rotate3D (shortcut notation)
  • Perspective (changes the perspective for a 3D view)

Using Transitions

Transitions describe functional and gradual state changes to objects on the web page and are possible with CSS3. Transitions are essentially a rudimentary form of animation. Any element on the page can have transitions applied but a common implementation is a hyperlink that appears as a tab which transitions into the page in order to be selected.

To create a transition effect, you must specfiy these:

  • the CSS property you want to add an effect to
  • the duration of the effect
  • this allows you to change properties values smoothly over a period of time.

Div {width :100px; height:100px background: red; -o-transition: width 2s; /* Opera*/ transition: width 2; }

Additional properties are:

  • Transition-timing-function - specifies the speed curve of the transition effect
  • Ease (default) specifies a slow start then fast, then end slow
  • Linear - transition same speed from start to end
  • Ease-in
  • Ease-out
  • Ease-in-out
  • Transition-delay
  • Transition-duration

Using Animations

CSS alows designers to apply animations to HTML object which describe how the element changes from one state to another. Animations also provide the ability to determine what the states of animation will be. Animations are considered as a series of keyframes each of which is a state the object will pass through over the course of its animation.

Animations are configured using the @ rule with the keyword keyframes. @-o-keyframes colorChange {from {background-color:red;} to {background-color:blue;} Then the animation is configured for a particular element by calling the name of the function using -o-animation:colorChange and specifying a timeframe.
